# Методы работы с сообщениями imconnector

---

## imconnector.delete.messages

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `CONNECTOR`, `LINE`, `MESSAGES` (все обязательные)

**Описание:**  
Удаляет сообщения из открытой линии, которые были переданы внешней системой.

> **Важно:** 
> - Метод работает только в контексте приложения (требуется OAuth авторизация)
> - Для удаления передавайте `message.id` и `chat.id`, которые использовали при отправке через `imconnector.send.messages`

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CONNECTOR` | string | Строковый код коннектора |
| `LINE` | integer | Идентификатор открытой линии |
| `MESSAGES` | array | Массив сообщений для удаления. Каждый элемент — объект с тремя обязательными блоками: `user`, `message`, `chat` |

**Структура объекта `MESSAGES[]`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `user` | object | Данные пользователя внешней системы |
| `message` | object | Данные сообщения внешней системы |
| `chat` | object | Данные чата или канала внешней системы |

**Объект `user`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор пользователя во внешней системе (обязательно) |

**Объект `message`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор сообщения во внешней системе, которое нужно удалить (обязательно) |

**Объект `chat`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор чата или канала во внешней системе (обязательно) |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "CONNECTOR": "myconnector",
    "LINE": 107,
    "MESSAGES": [
      {
        "user": {"id": "ext-user-42"},
        "message": {"id": "ext-msg-1001"},
        "chat": {"id": "channel-123"}
      }
    ]
  }' \
  https://your-domain.bitrix24.ru/rest/imconnector.delete.messages?auth=**put_access_token_here**
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "SUCCESS": true,
        "DATA": {
            "RESULT": [
                {
                    "user": "585",
                    "message": {
                        "id": "ext-msg-1001"
                    },
                    "chat": {
                        "id": "channel-123"
                    },
                    "SUCCESS": true
                }
            ]
        },
        "time": {...}
    }
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `SUCCESS` | boolean | `true` в случае успешного удаления сообщения |
| `DATA` | object | Данные обработки сообщений |
| `time` | time | Информация о времени выполнения запроса |

**Структура объекта `DATA`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `RESULT` | array | Массив результатов по каждому элементу `MESSAGES` |

**Структура элемента `RESULT[]`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `user` | string | Внутренний идентификатор пользователя Битрикс24 |
| `message` | object | Данные удаляемого сообщения |
| `chat` | object | Данные чата или канала |
| `SUCCESS` | boolean | Признак успешной обработки элемента |
| `ERRORS` | array | Массив текстов ошибок (при `SUCCESS = false`) |

**Структура объекта `message`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор сообщения во внешней системе |
| `date` | integer | Время сообщения в Unix Timestamp (если было передано) |
| `text` | string | Текст сообщения (если был передан) |
| `files` | array | Массив файлов сообщения (если были переданы) |

**Структура объекта `chat`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор чата или канала |
| `description` | string | Описание чата (возвращается, если передавали `url`) |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |
| 400 | `ERROR_ARGUMENT` | Не передан `CONNECTOR` |
| 400 | `ERROR_ARGUMENT` | Не передан `LINE` |
| 400 | `ERROR_ARGUMENT` | Не передан `MESSAGES` |
| 400 | `NOT_ACTIVE_LINE` | Линия с таким ID неактивна или не существует |
| 400 | `IMCONNECTOR_NO_CORRECT_PROVIDER` | Не удалось инициализировать провайдер для коннектора |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imconnector.update.messages

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `CONNECTOR`, `LINE`, `MESSAGES` (все обязательные)

**Описание:**  
Обновляет данные ранее переданных сообщений внешней системы в открытой линии.

> **Важно:** 
> - Метод работает только в контексте приложения (требуется OAuth авторизация)
> - Для обновления передавайте те же `message.id` и `chat.id`, которые использовали при отправке через `imconnector.send.messages`

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CONNECTOR` | string | Строковый код коннектора |
| `LINE` | integer | Идентификатор открытой линии |
| `MESSAGES` | array | Массив сообщений для обновления. Каждый элемент — объект с тремя обязательными блоками: `user`, `message`, `chat` |

**Структура объекта `MESSAGES[]`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `user` | object | Данные пользователя внешней системы |
| `message` | object | Данные сообщения внешней системы |
| `chat` | object | Данные чата внешней системы |

**Объект `user`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор пользователя во внешней системе (обязательно) |
| `last_name` | string | Фамилия пользователя |
| `name` | string | Имя пользователя |
| `picture` | object | Аватар: `{"url": "https://..."}` |
| `url` | string | Ссылка на профиль пользователя |
| `gender` | string | Пол: `male` или `female` |
| `email` | string | Email пользователя |
| `phone` | string | Телефон пользователя |
| `skip_phone_validate` | string | Отключить проверку телефона: `Y` |

**Объект `message`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор сообщения во внешней системе (которое нужно обновить) |
| `date` | integer | Время сообщения в Unix Timestamp (секунды) |
| `text` | string | Новый текст сообщения (обязательно передать `text` или `files`) |
| `files` | array | Массив файлов: `[{"url": "...", "name": "..."}]` |
| `disable_crm` | string | Отключить CRM трекер: `Y` |
| `user_id` | integer | Идентификатор менеджера в Битрикс24 |

**Объект `chat`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор чата или канала (обязательно) |
| `name` | string | Название чата или канала |
| `url` | string | Ссылка на чат или канал |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "CONNECTOR": "myconnector",
    "LINE": 107,
    "MESSAGES": [
      {
        "user": {
          "id": "ext-user-42",
          "last_name": "Иванов",
          "name": "Иван",
          "picture": {"url": "https://example.ru/u42.png"},
          "url": "https://example.ru/users/42",
          "gender": "male",
          "email": "ivan@example.ru",
          "phone": "+79990000000"
        },
        "message": {
          "id": "ext-msg-1001",
          "date": 1773266050,
          "text": "Добрый день, уточнили детали"
        },
        "chat": {
          "id": "channel-123",
          "name": "Канал поддержки",
          "url": "https://example.ru/chats/123"
        }
      }
    ]
  }' \
  https://your-domain.bitrix24.ru/rest/imconnector.update.messages?auth=**put_access_token_here**
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "SUCCESS": true,
        "DATA": {
            "RESULT": [
                {
                    "user": "585",
                    "message": {
                        "id": "ext-msg-1001",
                        "date": {...},
                        "text": "Добрый день, уточнили детали"
                    },
                    "chat": {
                        "id": "channel-123",
                        "name": "Канал поддержки",
                        "description": "Ссылка на исходный пост: https://example.ru/chats/123"
                    },
                    "SUCCESS": true
                }
            ]
        },
        "time": {...}
    }
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `SUCCESS` | boolean | `true` в случае успешного изменения сообщения |
| `DATA` | object | Данные обработки сообщений |
| `time` | time | Информация о времени выполнения запроса |

**Структура объекта `DATA`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `RESULT` | array | Массив результатов по каждому элементу `MESSAGES` |

**Структура элемента `RESULT[]`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `user` | string | Внутренний идентификатор пользователя Битрикс24 |
| `message` | object | Данные сообщения после обработки |
| `chat` | object | Данные чата после обработки |
| `extra` | object | Дополнительные флаги обработки |
| `SUCCESS` | boolean | Признак успешной обработки элемента |
| `ERRORS` | array | Массив текстов ошибок (при `SUCCESS = false`) |

**Структура объекта `message`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Внешний идентификатор сообщения |
| `date` | object | Дата сообщения после обработки |
| `text` | string | Текст сообщения |
| `files` | array | Массив файлов сообщения |

**Структура объекта `chat`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор чата или канала |
| `name` | string | Название чата или канала |
| `description` | string | Описание чата (например, ссылка на исходный пост) |

**Структура объекта `extra`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `skip_phone_validate` | string | Признак отключения проверки телефона |
| `disable_tracker` | string | Признак отключения CRM трекера |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |
| 400 | `ERROR_ARGUMENT` | Не передан `CONNECTOR` |
| 400 | `ERROR_ARGUMENT` | Не передан `LINE` |
| 400 | `ERROR_ARGUMENT` | Не передан `MESSAGES` |
| 400 | `NOT_ACTIVE_LINE` | Линия с таким ID неактивна или не существует |
| 400 | `IMCONNECTOR_NO_CORRECT_PROVIDER` | Не удалось инициализировать провайдер для коннектора |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imconnector.send.messages

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `CONNECTOR`, `LINE`, `MESSAGES` (все обязательные)

**Описание:**  
Принимает сообщения из внешней системы и передает их в открытую линию Битрикс24 через пользовательский коннектор.

> **Важно:** 
> - Метод работает только в контексте приложения (требуется OAuth авторизация)
> - В параметрах используются значения внешней системы: идентификатор пользователя, чата, ссылка на чат и его название

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CONNECTOR` | string | Строковый код коннектора (задается при регистрации через `imconnector.register`) |
| `LINE` | integer | Идентификатор открытой линии (можно получить через `imopenlines.config.get` или `imopenlines.config.list.get`) |
| `MESSAGES` | array | Массив сообщений. Каждый элемент — объект с тремя обязательными блоками: `user`, `message`, `chat` |

**Структура объекта `MESSAGES[]`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `user` | object | Данные пользователя внешней системы |
| `message` | object | Данные сообщения внешней системы |
| `chat` | object | Данные чата внешней системы |

**Объект `user`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор пользователя во внешней системе (обязательно) |
| `last_name` | string | Фамилия пользователя (макс. 25 символов) |
| `name` | string | Имя пользователя (макс. 25 символов) |
| `picture` | object | Аватар: `{"url": "https://..."}` |
| `url` | string | Ссылка на профиль пользователя |
| `gender` | string | Пол: `male` или `female` |
| `email` | string | Email пользователя |
| `phone` | string | Телефон пользователя |
| `skip_phone_validate` | string | Отключить проверку телефона: `Y` |

> **Примечание:** `name` и `last_name` могут содержать буквы, пробелы, дефисы и апострофы. Цифры и специальные символы использовать нельзя.

**Объект `message`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор сообщения во внешней системе |
| `date` | integer | Время сообщения в Unix Timestamp (секунды) |
| `text` | string | Текст сообщения (обязательно передать `text` или `files`) |
| `files` | array | Массив файлов: `[{"url": "...", "name": "..."}]` |
| `disable_crm` | string | Отключить CRM трекер: `Y` |
| `user_id` | integer | Идентификатор менеджера в Битрикс24 (отправка от имени менеджера) |

**Объект `chat`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор чата или канала (обязательно) |
| `name` | string | Название чата или канала |
| `url` | string | Ссылка на чат или канал |

> **Рекомендация:** Используйте те же значения `id`, `name`, `url`, что и в `DATA` метода `imconnector.connector.data.set`.

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "CONNECTOR": "myconnector",
    "LINE": 107,
    "MESSAGES": [
      {
        "user": {
          "id": "ext-user-42",
          "last_name": "Иванов",
          "name": "Иван",
          "picture": {"url": "https://example.ru/u42.png"},
          "url": "https://example.ru/users/42",
          "gender": "male",
          "email": "ivan@example.ru",
          "phone": "+79990000000",
          "skip_phone_validate": "Y"
        },
        "message": {
          "id": "ext-msg-1001",
          "date": 1773265993,
          "text": "Добрый день",
          "files": [
            {"url": "https://example.ru/files/spec.pdf", "name": "spec.pdf"}
          ],
          "disable_crm": "Y"
        },
        "chat": {
          "id": "channel-123",
          "name": "Канал поддержки",
          "url": "https://example.ru/chats/123"
        }
      }
    ]
  }' \
  https://your-domain.bitrix24.ru/rest/imconnector.send.messages?auth=**put_access_token_here**
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "SUCCESS": true,
        "DATA": {
            "RESULT": [
                {
                    "user": "585",
                    "message": {...},
                    "chat": {...},
                    "extra": {...},
                    "SUCCESS": true,
                    "session": {
                        "ID": "323",
                        "CHAT_ID": "1767"
                    }
                }
            ]
        },
        "time": {...}
    }
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `SUCCESS` | boolean | `true` в случае успешной отправки сообщения |
| `DATA` | object | Данные с информацией об отправленных сообщениях |
| `time` | time | Информация о времени выполнения запроса |

**Структура объекта `DATA`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `RESULT` | array | Массив результатов по каждому элементу `MESSAGES` |

**Структура элемента `RESULT[]`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `user` | string | Внутренний идентификатор пользователя Битрикс24 |
| `message` | object | Данные сообщения после обработки |
| `chat` | object | Данные чата после обработки |
| `extra` | object | Дополнительные флаги обработки |
| `SUCCESS` | boolean | Признак успешной обработки элемента |
| `ERRORS` | array | Массив текстов ошибок (при `SUCCESS = false`) |
| `session` | object | Информация о сессии открытой линии |

**Структура объекта `session`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `ID` | string | Идентификатор сессии открытой линии |
| `CHAT_ID` | string | Идентификатор чата в Битрикс24 |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |
| 400 | `ERROR_ARGUMENT` | Не передан `CONNECTOR` |
| 400 | `ERROR_ARGUMENT` | Не передан `LINE` |
| 400 | `ERROR_ARGUMENT` | Не передан `MESSAGES` |
| 400 | `ERROR_ARGUMENT` | `MESSAGES` передан не как массив |
| 400 | `ERROR_ARGUMENT` | Элементы `MESSAGES` переданы не как массивы |
| 400 | `ERROR_ARGUMENT` | В элементе `MESSAGES` отсутствуют `user`, `message` или `chat` |
| 400 | `NOT_ACTIVE_LINE` | Линия с таким ID неактивна или не существует |
| 400 | `IMCONNECTOR_NO_CORRECT_PROVIDER` | Не удалось инициализировать провайдер для коннектора |
| 400 | `IMCONNECTOR_NOT_SPECIFIED_CORRECT_COMMAND` | Не удалось определить команду обработки |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
