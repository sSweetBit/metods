# Методы статусов imconnector

---

## imconnector.send.status.delivery

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `CONNECTOR`, `LINE`, `MESSAGES` (все обязательные)

**Описание:**  
Подтверждает в Битрикс24, что исходящее сообщение открытой линии успешно доставлено во внешнюю систему.

> **Важно:**
> - Метод работает только в контексте приложения (требуется OAuth авторизация)
> - Метод не отправляет сообщение повторно, а только фиксирует результат доставки
> - При обработке статуса «доставлено» сообщение также помечается как «прочитано»
> - Метод не применяется для входящих сообщений из внешней системы

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CONNECTOR` | string | Строковый код коннектора |
| `LINE` | integer | Идентификатор открытой линии |
| `MESSAGES` | array | Массив статусов доставки. Каждый элемент — объект с блоками `im`, `message`, `chat` |

**Структура объекта `MESSAGES[]`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `im` | object | Внутренние идентификаторы сообщения в Битрикс24 |
| `message` | object | Данные сообщения во внешней системе |
| `chat` | object | Данные чата во внешней системе |

**Объект `im`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `chat_id` | integer | Идентификатор чата открытой линии в Битрикс24 |
| `message_id` | integer | Идентификатор сообщения в Битрикс24 |

> **Важно:** Поля `im.chat_id` и `im.message_id` получайте из события `OnImConnectorMessageAdd`, когда Битрикс24 отправляет сообщение во внешнюю систему. Внешние идентификаторы `message.id` и `chat.id` не заменяют `im.message_id` и `im.chat_id`.

**Объект `message`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | array | Массив внешних идентификаторов сообщений (например, `["ext-msg-1007"]`) |
| `date` | integer | Время доставки сообщения в Unix Timestamp (секунды) |

**Объект `chat`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | string | Идентификатор чата или канала во внешней системе |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "CONNECTOR": "myconnector",
    "LINE": 107,
    "MESSAGES": [
      {
        "im": {
          "chat_id": 323,
          "message_id": 85911
        },
        "message": {
          "id": ["ext-msg-1007"],
          "date": 1738065600
        },
        "chat": {
          "id": "channel-123"
        }
      }
    ]
  }' \
  https://your-domain.bitrix24.ru/rest/imconnector.send.status.delivery?auth=**put_access_token_here**
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "SUCCESS": true,
        "DATA": []
    },
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `SUCCESS` | boolean | `true`, если статус доставки успешно принят и обновлен |
| `DATA` | array | Пустой массив при успешной обработке |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |
| 400 | `ERROR_ARGUMENT` | Не передан `CONNECTOR` |
| 400 | `ERROR_ARGUMENT` | Не передан `LINE` |
| 400 | `ERROR_ARGUMENT` | Не передан `MESSAGES` |
| 400 | `IMCONNECTOR_NO_CORRECT_PROVIDER` | Не удалось инициализировать провайдер для коннектора |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
