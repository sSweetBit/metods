# Методы управления коннектором imconnector

---

## imconnector.list

**Scope:** `imopenlines`  
**Кто может выполнять:** пользователь с правом изменения коннекторов открытых линий  
**Параметры:** нет

**Описание:**  
Возвращает список всех коннекторов, которые зарегистрированы в Битрикс24.

> **Важно:** Метод работает только в контексте приложения (требуется OAuth авторизация).

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{}' \
  https://your-domain.bitrix24.ru/rest/imconnector.list?auth=**put_access_token_here**
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "livechat": "Онлайн-чат",
        "telegrambot": "Telegram",
        "network": "Битрикс24.Network",
        "myconnector": "Мой коннектор"
    },
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | object | Объект вида `connector_id: connector_name` для доступных коннекторов |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |
| 400 | `ACCESS_DENIED` | На портале не установлен модуль `imopenlines` |
| 400 | `ACCESS_DENIED` | У пользователя нет права изменения коннекторов |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imconnector.connector.data.set

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `CONNECTOR`, `LINE`, `DATA` (все обязательные)

**Описание:**  
Устанавливает настройки канала во внешней системе для пользовательского коннектора и указанной открытой линии.

> **Важно:** 
> - Метод работает только в контексте приложения (требуется OAuth авторизация)
> - Метод не поддерживает отдельную установку или изменение одного поля: настройки записываются и перезаписываются через объект `DATA` целиком

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CONNECTOR` | string | Строковый код коннектора (задается при регистрации через `imconnector.register`) |
| `LINE` | integer | Идентификатор открытой линии (можно получить через `imopenlines.config.get` или `imopenlines.config.list.get`) |
| `DATA` | object | Объект настроек канала во внешней системе |

**Структура объекта `DATA`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `ID` | string | Идентификатор канала во внешней системе (например, `channel-123`). Если готового нет, используйте уникальный ключ, например `myconnector_line107` |
| `URL` | string | Полная ссылка на чат или канал во внешней системе |
| `URL_IM` | string | Полная ссылка на чат в формате для интерфейса оператора. Если отдельной ссылки нет, используйте значение `URL` |
| `NAME` | string | Название канала для отображения в интерфейсе |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "CONNECTOR": "myconnector",
    "LINE": 107,
    "DATA": {
      "ID": "channel-123",
      "URL": "https://example.ru/chats/123",
      "NAME": "Канал поддержки"
    }
  }' \
  https://your-domain.bitrix24.ru/rest/imconnector.connector.data.set?auth=**put_access_token_here**
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {
        "start": 1738065600.11,
        "finish": 1738065600.20,
        "duration": 0.09,
        "processing": 0.04,
        "date_start": "2025-01-28T12:00:00+00:00",
        "date_finish": "2025-01-28T12:00:00+00:00"
    }
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если данные сохранены |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |
| 400 | `ERROR_ARGUMENT` | Не передан параметр `CONNECTOR` |
| 400 | `ERROR_ARGUMENT` | Не передан параметр `LINE` |
| 400 | `ERROR_ARGUMENT` | Не передан параметр `DATA` |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
