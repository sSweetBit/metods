# Отмена регистрации коннектора imconnector

---

## imconnector.unregister

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `ID` (обязательный)

**Описание:**  
Удаляет пользовательский коннектор (отменяет регистрацию).

> **Важно:** Метод работает только в контексте приложения (требуется OAuth авторизация).

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `ID` | string | Идентификатор коннектора, который был передан при регистрации в `imconnector.register` |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"ID":"myconnector"}' \
  https://your-domain.bitrix24.ru/rest/imconnector.unregister?auth=**put_access_token_here**
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "result": true
    },
    "time": {
        "start": 1738065600.11,
        "finish": 1738065600.25,
        "duration": 0.14,
        "processing": 0.06,
        "date_start": "2025-01-28T12:00:00+00:00",
        "date_finish": "2025-01-28T12:00:00+00:00"
    }
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если регистрация коннектора снята |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

Метод может возвращать ошибки в двух форматах:

1. **HTTP 200** с `result = false`:

| Код | Описание |
|-----|----------|
| `APPLICATION_UNREGISTRATION_ERROR` | Не удалось удалить коннектор по ID для текущего приложения |
| `NO_APPLICATION_ID` | В контексте запроса не найдено приложение |

2. **HTTP 403** для системной ошибки авторизации:

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
