# Коннекторы открытых линий: обзор методов imconnector

**Коннекторы открытых линий** — это инструмент для интеграции внешних мессенджеров и социальных сетей с Битрикс24.

Коннекторы позволяют:
- Отвечать на сообщения клиентов из разных каналов в едином интерфейсе
- Подключать чат-ботов для автоматизации общения с клиентами
- Анализировать статистику диалогов для оценки эффективности коммуникаций

> **Важно:** Методы `imconnector.*` в текущей версии не поддерживают работу через вебхуки. Требуется OAuth авторизация в контексте приложения.

## Связь коннекторов с другими объектами

| Объект | Описание |
|--------|----------|
| **Открытые линии** | Собирают сообщения с коннекторов, управляют очередями и распределяют сообщения между сотрудниками. Работать с открытыми линиями следует с помощью группы методов `imopenlines.*` |
| **CRM** | На основе сообщений можно автоматически создавать лиды и сделки методами `crm.lead.add` и `crm.deal.add` |
| **Пользователь** | Получить идентификатор пользователя позволяют методы `user.get` и `user.search` |
| **Чат** | За переписку с пользователями отвечают методы `im.chat.*` и `im.message.*` |
| **Чат-боты** | Настройка автоматических ответов и чат-ботов выполняется с помощью методов `imbot.*` |

## Как подключить коннектор

1. **Зарегистрируйте коннектор** с помощью метода `imconnector.register`
2. **Активируйте его** методом `imconnector.activate`
3. **Установите данные коннектора** методом `imconnector.connector.data.set`
4. **Проверьте статус коннектора** методом `imconnector.status` и убедитесь, что он готов к работе

## Как работать с сообщениями

| Действие | Метод |
|----------|-------|
| Отправить сообщение | `imconnector.send.messages` |
| Изменить отправленные сообщения | `imconnector.update.messages` |
| Удалить сообщения | `imconnector.delete.messages` |
| Обновить статус «доставлено» | `imconnector.send.status.delivery` |

## Как добавить виджет в Контакт-центр

Чтобы добавить виджет коннектора в Контакт-центр, используйте код встройки `CONTACT_CENTER`. Этот код необходимо указать в параметре `PLACEMENT` метода `placement.bind`.

## Обработка ошибок

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `ERROR_ARGUMENT` | Обязательный параметр не передан или передан в неверном формате |
| 400 | `NOT_ACTIVE_LINE` | Линия с таким ID неактивна или не существует |
| 400 | `IMCONNECTOR_NO_CORRECT_PROVIDER` | Не удалось инициализировать провайдер для коннектора |
| 403 | `WRONG_AUTH_TYPE` | Метод вызван не в контексте приложения (требуется OAuth) |
| 403 | `ACCESS_DENIED` | Недостаточно прав для выполнения действия |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
