# Подразделения в чатах: обзор методов im.department

Методы `im.department.*` получают информацию о пользователях подразделений из структуры компании.

> **Важно:** Методы работают только для сотрудников компании (интранет-пользователей). Методы недоступны для экстранет-пользователей и ботов.

## Связь методов с другими объектами

**Подразделение.** Методы получают информацию о пользователях по идентификатору департамента `ID`. 

Получить идентификатор департамента можно:
- Методом получения списка подразделений
- Методом поиска подразделений по названию

## Получить данные о подразделении

Метод `im.department.get` получает название подразделения и идентификатор руководителя. С помощью параметра `USER_DATA = 'Y'` в ответ можно добавить данные о руководителе подразделения.

## Получить список пользователей

Получить списки идентификаторов пользователей можно методами:

| Метод | Описание |
|-------|----------|
| `im.department.managers.get` | Руководители указанных подразделений |
| `im.department.employees.get` | Сотрудники указанных подразделений |
| `im.department.colleagues.list` | Список коллег текущего пользователя. Для руководителя метод вернет список подчиненных и всех руководителей |

C помощью параметра `USER_DATA = 'Y'` можно получить подробные данные по каждому сотруднику.

## Обзор методов

| Метод | Описание |
|-------|----------|
| [im.department.get](#imdepartmentget) | Получает информацию о подразделении |
| [im.department.colleagues.list](#imdepartmentcolleagueslist) | Получает список коллег текущего пользователя |
| [im.department.managers.get](#imdepartmentmanagersget) | Получает список руководителей подразделений |
| [im.department.employees.get](#imdepartmentemployeesget) | Получает список сотрудников подразделений |

## Общие параметры

Большинство методов группы поддерживают параметр `USER_DATA`:

| Значение | Описание |
|----------|----------|
| `Y` | Возвращать подробные данные пользователей (объекты с полной информацией) |
| `N` | Возвращать только идентификаторы пользователей |

## Обработка ошибок

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `ID_EMPTY` | Параметр `ID` не передан, передан неверно или пустым |
| 400 | `INVALID_FORMAT` | Параметр `ID` передан в неверном формате |
| 403 | `ACCESS_ERROR` | Метод недоступен для экстранет-пользователей и ботов |

Общие ошибки: см. [Обработка ошибок](../../common/errors.md)

## im.department.get

**Scope:** `im`  
**Кто может выполнять:** любой интранет-пользователь, кроме ботов  
**Параметры:** `ID` (обязательный), `USER_DATA` (опциональный)

**Описание:**  
Получает данные о подразделениях по их идентификаторам.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `ID` | array | Массив идентификаторов подразделений |
| `USER_DATA` | string | Возвращать данные руководителя: `Y` — да, `N` — нет |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"ID":[3,7],"USER_DATA":"Y"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/im.department.get.json
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | array | Список подразделений |
| `time` | time | Информация о времени выполнения запроса |

**Структура объекта `department`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | integer | Идентификатор подразделения |
| `name` | string | Название подразделения |
| `full_name` | string | Полное название в структуре компании |
| `manager_user_id` | integer | ID руководителя подразделения |
| `manager_user_data` | object | Данные руководителя (только при `USER_DATA = 'Y'`) |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `INVALID_FORMAT` | Параметр `ID` не передан, передан неверно или пустым |
| 403 | `ACCESS_ERROR` | Метод недоступен для экстранет-пользователей и ботов |

Общие ошибки: см. [Обработка ошибок](../../common/errors.md)

## im.department.colleagues.list

**Scope:** `im`  
**Кто может выполнять:** любой интранет-пользователь, кроме ботов  
**Параметры:** `USER_DATA` (опциональный), `OFFSET` (опциональный), `LIMIT` (опциональный)

**Описание:**  
Получает список коллег текущего пользователя. Для руководителя метод вернет список подчиненных и всех руководителей.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `USER_DATA` | string | Возвращать подробные данные: `Y` — да, `N` — нет |
| `OFFSET` | integer | Смещение выборки пользователей |
| `LIMIT` | integer | Количество элементов (по умолчанию 10, макс. 50) |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"USER_DATA":"Y","OFFSET":0,"LIMIT":5}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/im.department.colleagues.list.json
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | array | Список пользователей (ID или объекты) |
| `total` | integer | Общее количество пользователей |
| `next` | integer | Смещение для следующей страницы |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 403 | `ACCESS_ERROR` | Метод недоступен для экстранет-пользователей и ботов |

Общие ошибки: см. [Обработка ошибок](../../common/errors.md)
