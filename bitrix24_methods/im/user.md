# Пользователи в чатах: обзор методов im.user

Для работы с пользователями в чатах используйте группу методов `im.user.*`.

## Связь методов с другими объектами

**Пользователь.** Методы получают информацию о пользователях по идентификатору `ID`. 

Узнать идентификаторы пользователей чата можно с помощью метода `im.chat.user.list`.

## Информация о пользователях

Метод `im.user.get` получает информацию о текущем пользователе или конкретном пользователе, если заполнить параметр `ID`.

Получить информацию о нескольких пользователях можно методом `im.user.list.get`. В параметре `ID` передайте массив с идентификаторами пользователей, например, `ID: [3, 5, 7]`.

## Статус пользователя в чатах

Метод `im.user.status.set` устанавливает статус пользователя в чате:

| Статус | Описание |
|--------|----------|
| `online` | Пользователь получает все уведомления |
| `dnd` | Режим «Не беспокоить», пользователь не получает уведомления о новых сообщениях |

Узнать статус можно методом `im.user.status.get`.

> **Важно:** Методы работают только для текущего пользователя.

## Обзор методов

| Метод | Описание |
|-------|----------|
| [im.user.get](#imuserget) | Получает данные о пользователе |
| [im.user.list.get](#imuserlistget) | Получает данные о списке пользователей |
| [im.user.status.get](#imuserstatusget) | Получает информацию об установленном статусе пользователя |
| [im.user.status.idle.start](#imuserstatusidlestart) | Устанавливает автоматический статус «Отошел» |
| [im.user.status.idle.end](#imuserstatusidleend) | Отключает автоматический статус «Отошел» |

## Методы для предыдущей версии чата

> **Примечание:** Методы разработаны для предыдущей версии чата. В текущей версии чата М1 они работают, но результаты не отображаются в интерфейсе.

| Метод | Описание |
|-------|----------|
| `im.user.status.idle.start` | Устанавливает автоматический статус «Отошел» |
| `im.user.status.idle.end` | Отключает автоматический статус «Отошел» |

## Обработка ошибок

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `ID_EMPTY` | Параметр `ID` не передан, передан неверно или пустым |
| 400 | `USER_NOT_EXISTS` | Пользователь с указанным ID не найден |
| 400 | `INVALID_FORMAT` | Параметр передан в неверном формате |
| 403 | `ACCESS_DENIED` | Экстранет-пользователь запрашивает пользователя не из своей группы |

Общие ошибки: см. [Обработка ошибок](../../common/errors.md)

## im.user.get

**Scope:** `im`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `ID` (опциональный), `AVATAR_HR` (опциональный)

**Описание:**  
Получает данные о текущем пользователе или о пользователе по ID.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `ID` | integer | Идентификатор пользователя. Если не передан, метод вернет данные текущего пользователя |
| `AVATAR_HR` | string | Запрос аватара в высоком разрешении: `Y` или `N` (по умолчанию `N`) |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"ID":5,"AVATAR_HR":"Y"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/im.user.get.json
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | object | Объект с данными пользователя |
| `time` | time | Информация о времени выполнения запроса |

**Основные поля `result`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `id` | integer | Идентификатор пользователя |
| `name` | string | Имя и фамилия |
| `work_position` | string | Должность |
| `avatar` | string | Ссылка на аватар |
| `gender` | string | Пол |
| `email` | string | Email |
| `status` | string | Статус (в новом мессенджере всегда `online`) |
| `departments` | array | Массив ID подразделений |

> **Примечание:** Для проверки установленного статуса используйте `im.user.status.get`, для изменения — `im.user.status.set`.

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `ID_EMPTY` | Передан `ID` <= 0 |
| 400 | `USER_NOT_EXISTS` | Пользователь с указанным ID не найден |
| 400 | `ACCESS_DENIED` | Экстранет-пользователь запрашивает пользователя не из своей группы |

Общие ошибки: см. [Обработка ошибок](../../common/errors.md)

## im.user.list.get

**Scope:** `im`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `ID` (обязательный), `AVATAR_HR` (опциональный), `RESULT_TYPE` (опциональный)

**Описание:**  
Возвращает данные о пользователях по списку идентификаторов.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `ID` | array | Массив идентификаторов пользователей или JSON-строка с массивом |
| `AVATAR_HR` | string | Запрос аватара в высоком разрешении: `Y` или `N` |
| `RESULT_TYPE` | string | Формат результата: `array` — массив объектов, любое другое — объект с ключами-ID |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"ID":[4,5],"AVATAR_HR":"Y","RESULT_TYPE":"array"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/im.user.list.get.json
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | array\|object | Данные пользователей (массив при `RESULT_TYPE=array`, иначе объект с ключами-ID) |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `INVALID_FORMAT` | Параметр `ID` не передан или передан в неверном формате |

Общие ошибки: см. [Обработка ошибок](../../common/errors.md)

## im.user.status.idle.start

**Scope:** `im`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `AGO` (опциональный)

**Описание:**  
Включает автоматический статус «Отошел» для текущего пользователя.

> **Примечание:** Метод разработан для предыдущей версии чата. В текущей версии чата М1 он работает, но результаты не отображаются в интерфейсе.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `AGO` | integer | Сколько минут назад установить статус «Отошел». По умолчанию 10. Минимальное значение 1 |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"AGO":10}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/im.user.status.idle.start.json
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если статус установлен |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

Специфичные ошибки метода: отсутствуют

Общие ошибки: см. [Обработка ошибок](../../common/errors.md)
