## imopenlines.session.intercept

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь с правами на диалог  
**Параметры:** `CHAT_ID` (обязательный)

**Описание:**  
Переводит текущий диалог на оператора, который вызывает метод (забрать диалог у текущего оператора).

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CHAT_ID` | integer | Идентификатор чата открытой линии |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"CHAT_ID":2043}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.session.intercept.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если диалог успешно переведен на текущего оператора |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `CHAT_ID_EMPTY` | Не передан `CHAT_ID` или значение <= 0 |
| 400 | `OPERATOR_WRONG` | Операция недоступна для текущего пользователя или состояния диалога |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imopenlines.session.open

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь с правом на доступ к диалогу  
**Параметры:** `USER_CODE` (обязательный)

**Описание:**  
Возвращает идентификатор чата открытой линии по коду пользователя USER_CODE.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `USER_CODE` | string | Строковый код пользователя. Формат: `<connector>|<LINE_ID>|<CONNECTOR_CHAT_ID>|<CONNECTOR_USER_ID>` |

> **Формат USER_CODE:**
> - `<connector>` — идентификатор коннектора: livechat, telegram и другие
> - `<LINE_ID>` — идентификатор открытой линии
> - `<CONNECTOR_CHAT_ID>` — идентификатор чата в канале
> - `<CONNECTOR_USER_ID>` — идентификатор пользователя в канале

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"USER_CODE":"livechat|22|1761|587"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.session.open.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "chatId": "1763"
    },
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | object | Корневой объект ответа |
| `time` | time | Информация о времени выполнения запроса |

**Структура объекта `result`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `chatId` | integer | Идентификатор чата открытой линии |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `ACCESS_DENIED` | Нет доступа к чату по указанному `USER_CODE` |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imopenlines.session.history.get

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь  
**Параметры:** один из `SESSION_ID` / `CHAT_ID`

**Описание:**  
Возвращает историю сообщений сессии открытой линии.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `SESSION_ID` | integer | Идентификатор сессии (можно получить из `imopenlines.dialog.get`, поле `entity_data_1`) |
| `CHAT_ID` | integer | Числовой идентификатор чата (без префикса `chat`) |

> **Примечание:** Если передан `SESSION_ID`, работает по нему. Если не передан — пытается определить последнюю сессию по `CHAT_ID`. Рекомендуется передавать `SESSION_ID`.

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"SESSION_ID":321}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.session.history.get.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "chatId": 1763,
        "canJoin": "Y",
        "sessionId": 321,
        "message": {...},
        "users": {...},
        "chat": {...},
        "files": {...}
    },
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | object | Корневой объект ответа |
| `time` | time | Информация о времени выполнения запроса |

**Основные поля `result`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `chatId` | integer | Идентификатор чата |
| `sessionId` | integer | Идентификатор сессии |
| `canJoin` | string | Флаг возможности подключения (`Y`/`N`) |
| `canVoteHead` | string | Флаг возможности оценки руководителем |
| `message` | object | Сообщения чата (индексированы по ID) |
| `users` | object | Участники чата |
| `chat` | object | Данные чата |
| `files` | object | Файлы сообщений |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `MISSING_REQUIRED_PARAM` | Не переданы `SESSION_ID` и `CHAT_ID` |
| 400 | `INVALID_SESSION_ID` | Не удалось определить сессию |
| 400 | `ACCESS_DENIED` | Недостаточно прав для просмотра истории |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imopenlines.session.head.vote

**Scope:** `imopenlines`  
**Кто может выполнять:** руководитель с правом оценки в настройках линии  
**Параметры:** `SESSION_ID` (обязательный), один из `RATING` / `COMMENT`

**Описание:**  
Сохраняет оценку руководителя и комментарий по завершенному диалогу.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `SESSION_ID` | integer | Идентификатор сессии (можно получить через `imopenlines.session.history.get`) |
| `RATING` | integer | Оценка руководителя (от 1 до 5) |
| `COMMENT` | string | Комментарий руководителя к оценке |

> **Примечание:** Необходимо передать хотя бы один из параметров: `RATING` или `COMMENT`.

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"SESSION_ID":1743,"RATING":5,"COMMENT":"Отличная обработка"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.session.head.vote.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если оценка сохранена |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `EMPTY_SESSION_ID` | Не передан `SESSION_ID` или значение <= 0 |
| 400 | `EMPTY_VOTE_PARAMS` | Не переданы одновременно `RATING` и `COMMENT` |
| 400 | `ACCESS_DENIED` | Текущий пользователь не может оценить сессию |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
