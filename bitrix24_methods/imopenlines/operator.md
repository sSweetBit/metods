# Методы оператора imopenlines

---

## imopenlines.operator.answer

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь с правами на диалог  
**Параметры:** `CHAT_ID` (обязательный)

**Описание:**  
Передает диалог открытой линии текущему оператору (забрать диалог себе).

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CHAT_ID` | integer | Идентификатор чата открытой линии (можно получить через `imopenlines.session.open` или `imopenlines.dialog.get`) |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"CHAT_ID":2043}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.operator.answer.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если оператор успешно забрал диалог |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `CHAT_ID` | Указан не корректный идентификатор чата |
| 400 | `CHAT_TYPE` | Указанный чат не является открытой линией |
| 400 | `ACCESS_DENIED` | Недостаточно прав на диалог |
| 400 | `USER_ID` | Не определен пользователь |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imopenlines.operator.transfer

**Scope:** `imopenlines`  
**Кто может выполнять:** любой пользователь с правами на диалог  
**Параметры:** `CHAT_ID` (обязательный), один из `TRANSFER_ID` / `USER_ID` / `QUEUE_ID`

**Описание:**  
Передает диалог другому оператору или в очередь другой открытой линии.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CHAT_ID` | integer | Идентификатор чата открытой линии |
| `TRANSFER_ID` | string\|integer | Универсальный параметр: `USER_ID` или строка `queue<QUEUE_ID>` |
| `USER_ID` | integer | Идентификатор оператора |
| `QUEUE_ID` | integer | Идентификатор линии |

> **Примечание:** Рекомендуется передавать только один параметр назначения.

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"CHAT_ID":2043,"USER_ID":15}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.operator.transfer.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если диалог успешно передан |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `CHAT_ID_EMPTY` | Не передан `CHAT_ID` или значение <= 0 |
| 400 | `USER_ID_EMPTY` | Передан пустой или некорректный `USER_ID` |
| 400 | `QUEUE_ID_EMPTY` | Передан пустой или некорректный `QUEUE_ID` |
| 400 | `TRANSFER_ID_EMPTY` | Не переданы параметры назначения |
| 400 | `OPERATOR_WRONG` | Целевой оператор или линия недоступны |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
