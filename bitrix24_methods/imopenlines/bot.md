# Методы ботов imopenlines

---

## imopenlines.bot.session.operator

**Scope:** `imopenlines`, `imbot`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `CHAT_ID` (обязательный)

**Описание:**  
Переключает диалог со стороны чат-бота на свободного оператора открытой линии.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CHAT_ID` | integer | Идентификатор чата (можно получить через `imopenlines.dialog.get` или `imopenlines.crm.chat.get`) |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"CHAT_ID":112}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.bot.session.operator.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если диалог успешно передан свободному оператору |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `CHAT_ID_EMPTY` | Не передан `CHAT_ID` или передано значение <= 0 |
| 400 | `WRONG_CHAT` | Указанный диалог уже ведет оператор, а не чат-бот |
| 403 | `WRONG_AUTH_TYPE` | Метод вызван с сессионной авторизацией (запрещено) |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imopenlines.bot.session.message.send

**Scope:** `imopenlines`, `imbot`  
**Кто может выполнять:** любой пользователь  
**Параметры:** `CHAT_ID` (обязательный), `NAME` (опциональный), `MESSAGE` (опциональный)

**Описание:**  
Отправляет в диалог открытой линии автоматическое сообщение от чат-бота приложения.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CHAT_ID` | integer | Идентификатор чата для отправки сообщения |
| `NAME` | string | Код типа автоответа: `WELCOME` — приветствие из настроек, `DEFAULT` — текст из `MESSAGE` |
| `MESSAGE` | string | Текст сообщения для режима `NAME=DEFAULT` |

> **Примечание:** Если передать `NAME=DEFAULT` и пустой `MESSAGE`, сообщение не добавится.

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"CHAT_ID":112,"NAME":"DEFAULT","MESSAGE":"Здравствуйте! Чем могу помочь?"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.bot.session.message.send.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | Флаг успешного вызова (не гарантирует создание сообщения) |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `CHAT_ID_EMPTY` | Не передан `CHAT_ID` или передано значение <= 0 |
| 403 | `WRONG_AUTH_TYPE` | Метод вызван с сессионной авторизацией (запрещено) |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)

## imopenlines.bot.session.transfer

**Scope:** `imopenlines`, `imbot`  
**Кто может выполнять:** пользователь приложения с зарегистрированным чат-ботом  
**Параметры:** `CHAT_ID` (обязательный), один из `USER_ID` / `QUEUE_ID` / `TRANSFER_ID`, `LEAVE` (опциональный)

**Описание:**  
Переводит диалог открытой линии от чат-бота на указанного оператора или очередь.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `CHAT_ID` | integer | Идентификатор чата |
| `USER_ID` | integer | Идентификатор сотрудника |
| `QUEUE_ID` | integer | Идентификатор очереди |
| `TRANSFER_ID` | string\|integer | Универсальный параметр: `USER_ID` или `queue<QUEUE_ID>` |
| `LEAVE` | boolean | Флаг присутствия бота: `Y` — покидает сразу, `N` — остается до подтверждения |

> **Примечание:** Приоритет параметров: `USER_ID` → `QUEUE_ID` → `TRANSFER_ID`.

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"CHAT_ID":112,"USER_ID":12,"LEAVE":"N"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/imopenlines.bot.session.transfer.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": true,
    "time": {...}
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | boolean | `true`, если диалог успешно переведен |
| `time` | time | Информация о времени выполнения запроса |

**Обработка ошибок:**

| Статус | Код | Описание |
|--------|-----|----------|
| 400 | `CHAT_ID_EMPTY` | Не передан `CHAT_ID` или значение <= 0 |
| 400 | `USER_ID_EMPTY` | Передан пустой `USER_ID` |
| 400 | `QUEUE_ID_EMPTY` | Передан пустой `QUEUE_ID` |
| 400 | `TRANSFER_ID_EMPTY` | Не переданы параметры назначения |
| 400 | `BOT_ID_ERROR` | В приложении не найден зарегистрированный чат-бот |
| 400 | `OPERATOR_WRONG` | Перевод недоступен |
| 403 | `WRONG_AUTH_TYPE` | Метод вызван с сессионной авторизацией |

Общие ошибки: см. [Обработка ошибок](../common/errors.md)
