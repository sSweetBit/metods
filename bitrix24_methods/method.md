# Методы проверки методов

---

## method.get

**Scope:** базовый  
**Кто может выполнять:** любой пользователь  
**Параметры:** `name` (обязательный)

**Описание:**  
Возвращает два параметра `isExisting` и `isAvailable`, которые определяют существование метода на портале и его доступность для вызова.

**Параметры метода:**

| Название | Тип | Описание |
|----------|-----|----------|
| `name` | string | Название метода для проверки в нижнем регистре (например, `user.get`) |

**Пример запроса:**
```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{"name": "user.get"}' \
  https://your-domain.bitrix24.ru/rest/1/webhook_key/method.get.json
```

**Пример ответа (HTTP 200):**
```json
{
    "result": {
        "isExisting": true,
        "isAvailable": true
    },
    "time": {
        "start": 1721983189.4629,
        "finish": 1721983189.50346,
        "duration": 0.0405528545379639,
        "processing": 0.000152111053466796,
        "date_start": "2024-07-26T08:39:49+00:00",
        "date_finish": "2024-07-26T08:39:49+00:00",
        "operating": 0
    }
}
```

**Возвращаемые данные:**

| Название | Тип | Описание |
|----------|-----|----------|
| `result` | object | Объект с параметрами проверки метода |
| `time` | time | Информация о времени выполнения запроса |

**Структура объекта `result`:**

| Название | Тип | Описание |
|----------|-----|----------|
| `isExisting` | boolean | `true` — метод существует на этом портале |
| `isAvailable` | boolean | `true` — метод доступен для вызова с текущими правами (scope) приложения |

**Обработка ошибок:**

Специфичные ошибки метода: отсутствуют

Общие ошибки: см. [Обработка ошибок](common/errors.md)
